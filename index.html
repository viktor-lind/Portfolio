onePlant

Oneplant is a semi-automatic indoor-irrigation system. The user takes care of the main plant while knowing that every other plant connected to the system will get the right amount of water. By monitoring the moisture levels in the soil we can determine if the plant needs water, thus eliminating the risk of forgetting to water or over water the plant. The hub-plant will signal when water level is getting to low. The sub-plants connected to the main plant will signal if they need additional water since every plant requires different amounts of water. The application will work as an assistant with additional information about each plant where the user can select which type of plant they have connected to the system. The application will not assist in the watering process since we want to keep the feeling of taking care of the plants and not make it fully automatic. Although it will warn if water levels are getting to low or the user forgets to water the plants.

Ro

Ro is based on the charachsteristics of a spinning top. The game aims to let the player feel the same satisfaction one would get when spinning a physical top. The player picks up the digital artifact by touching it. A swing-motion will initialize the rotation of the orbs surounding the artifact. By letting go of the screen the player can controll the artifact by tilting the mobile device. Ro is made for mobile devices with Unity 3D. The game was user tested at an early stage, the end result is based on those tests.
My main task was programming the game.

Got more experience with C#
A lot of practice working with user research

Kronox schedule

This project is a redesign of the schedules displayed on the monitors located at the entrence of Malmö Högskola. Based on user research we found that the existing scheduled lacked functionality. It was hard to find the courses and in which classroom they were scheduled. So we did a total redesign shown in the image.
